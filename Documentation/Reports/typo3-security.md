# TYPO3 Security Audit (v14.3 LTS)

Date: 2026-05-03
Risk level: Medium

## Summary
Desiderio is a presentation-only theme/component package: no controllers, no AJAX endpoints, no DataHandler hooks, no PSR-14 listeners, no auth flow. Most attack surface is editor-controlled HTML/URL fields rendered into Fluid templates. Risk is concentrated in (a) inline `<script>`/`style="..."` blocks across charts and a few atoms (CSP-incompatible, plus a small XSS surface via editor-controlled iframe URLs and unvalidated numeric/style fields), and (b) `f:format.raw` / `f:format.html` on `bodytext`-style fields. The seed CLI command (Classes/Command/SeedStyleguidePagesCommand.php) is well-parameterised and CLI-only by design (no HTTP route), but it writes broadly to `pages`, `tt_content`, `sys_file_reference`, and `sys_file_metadata` and must remain CLI-gated in production. No SQLi, no JSON-deserialization, no secrets, no PII logging found.

## Findings (severity-tagged)

- [HIGH] Editor-controlled iframe `src` without host allowlist — Free-text `Text` field rendered straight into `<iframe src>`; an editor (compromised or careless) can embed arbitrary origins (drive-by, click-jacking, tracking). Path: `ContentBlocks/ContentElements/map-embed/templates/frontend.html:18`, `ContentBlocks/ContentElements/video-embed/templates/frontend.html:18-19`, `ContentBlocks/ContentElements/feature-video/templates/frontend.html:22-23`, `ContentBlocks/ContentElements/testimonial-video/templates/frontend.html:24`. Fix: validate `embed_url`/`video_url` against an allowlist (youtube.com, youtube-nocookie.com, vimeo.com, player.vimeo.com, OSM, Google Maps embed) in TCA `eval` or a dedicated PSR-14 DataHandler `processDatamap_*` listener; add `sandbox`/`allow` and a `Content-Security-Policy: frame-src` rule.
- [HIGH] Unsanitised editor input flows into `<iframe>`'s wrapping `style="height: …px"` — `data.height` is a free `Text` field interpolated into the `style` attribute via `f:or(alternative: '400')` without numeric coercion or escaping; an editor entering `0px} body{display:none}` breaks the page or breaks out of style. Path: `ContentBlocks/ContentElements/map-embed/templates/frontend.html:17`. Fix: cast in template (`{data.height -> f:format.number()}`) or change TCA to `Number`.
- [HIGH] CSP-blocking inline `<script>` in 11 chart elements — Each chart inlines a self-executing IIFE that interpolates `{data.uid}` directly into JS literal context (e.g. `getElementById('chart-' + {data.uid})` and `const uid = {data.uid};`). Although `uid` is an integer column so injection is impractical, every block is still CSP-incompatible (no nonce, inline). Paths: `ContentBlocks/ContentElements/chart/templates/frontend.html:43-113`, `chart-bar/templates/frontend.html:34-119`, `chart-area`, `chart-line`, `chart-pie`, `chart-donut`, `chart-radar`, `chart-sparkline`, `chart-stacked-bar/templates/frontend.html:30-92`, `chart-heatmap/templates/frontend.html:28+`, `chart-contribution/templates/frontend.html:67+`, `metric-dashboard/templates/frontend.html:45+`. Fix: move chart bootstrap into `Resources/Public/Js/desiderio.js` with `data-*` attributes (already partially done), or emit nonces via `f:asset.script` / TYPO3 `assetCollector`.
- [HIGH] Inline `style="--gradient-from: {data.gradient_from}; --gradient-to: {data.gradient_to}"` — Editor `gradient_from`/`gradient_to` Text fields injected unescaped into `style`. Even with `f:or` fallback, an editor entering `red; background-image: url(//attacker)` injects a CSS rule. Path: `ContentBlocks/ContentElements/hero-gradient/templates/frontend.html:8`. Fix: validate via TCA `eval=trim,unicode` plus a regex allowlist (`^(var\(--[a-z0-9-]+\)|#[0-9a-fA-F]{3,8}|[a-z]+)$`), or move to a fixed CSS variable and only allow editor to pick a Select.
- [MEDIUM] `chart-stacked-bar` builds DOM with `innerHTML` from JS-side template literal — `item.innerHTML = '<span … style="background:' + colors[j…] + '"></span> S' + (j+1)`; `colors` is a hard-coded array, but the pattern is brittle and CSP-noisy. Path: `ContentBlocks/ContentElements/chart-stacked-bar/templates/frontend.html:60`. Fix: build via `document.createElement` like sibling charts.
- [MEDIUM] `f:format.html` on `newsItem.bodytext`/`newsItem.teaser` — Renders RTE-stored HTML through TYPO3's parseFunc. This is the standard EXT:news pattern, but downgrades safety to the htmlSanitizer config. If site disables sanitizer or relaxes `lib.parseFunc_RTE`, stored XSS becomes possible. Paths: `Resources/Private/Extensions/News/Templates/News/Detail.html:74,87`, `Resources/Private/Extensions/News/Partials/List/Item.html:58-59`, `Resources/Private/Extensions/News/Templates/News/MagazineList.html:69-70`. Fix: ensure TYPO3 14 `htmlSanitizer` is enabled site-wide and document the dependency in README.
- [MEDIUM] `f:format.raw` on FluidStyledContent HTML CType — `{record.bodytext -> f:format.raw()}` outputs raw HTML stored in CType `html`. This matches core behaviour (the `html` CType is a documented backend-restricted feature), but if BE editors without `tt_content.CType:html` privileges can be elevated, stored XSS lands. Path: `Resources/Private/FluidStyledContent/Templates/Html.fluid.html:12`. Fix: keep the CType `html` permission restricted to admins via TSconfig (`TCEFORM.tt_content.CType.removeItems := addToList(html)` for non-admin groups).
- [MEDIUM] `f:format.raw` on Solr "explain"/debug output — `<f:format.raw>{documentScoreAnalyze}</f:format.raw>` renders HTML from EXT:solr's debug partial. Only visible when `plugin.tx_solr.logging.debugOutput` is on. Combined with `Configuration/Sets/SolrDefaults/setup.typoscript:6` which explicitly sets `plugin.tx_solr.logging.debugOutput = 0`, exposure is gated. Paths: `Resources/Private/Solr/Partials/Result/Document.html:61`, `Resources/Private/Extensions/Solr/Partials/Result/Document.html:60`. Fix: add a `<f:if condition="{f:debug()}">` or a TypoScript-driven flag so it can never render in production even if debug accidentally toggles on.
- [MEDIUM] Unvalidated numeric editor input flows into inline `style="opacity"` and `style="width"` — Same shape as map-embed height; `overlay_opacity` and `progress_value` come from editor and are interpolated into the `style` attribute. Paths: `ContentBlocks/ContentElements/hero-fullscreen/templates/frontend.html:12`, `ContentBlocks/ContentElements/hero/templates/frontend.html:58`, `ContentBlocks/ContentElements/progress-bar/templates/frontend.html:33,35`, `ContentBlocks/ContentElements/stats-bar/templates/frontend.html:32`. Fix: pipe through `f:format.number()` or coerce via TCA `eval=int`.
- [MEDIUM] `style` argument passed through to atoms is an unescaped pass-through — `Resources/Private/Components/Atom/{Skeleton,AspectRatio,ScrollArea}/*.fluid.html` and `Resources/Private/Components/Layout/Section/Section.fluid.html:30` accept a `style` argument and write it raw into the DOM. Currently only callers inside the package use these and pass safe values, but a future caller passing editor data inherits the risk. Paths: `Resources/Private/Components/Atom/Skeleton/Skeleton.fluid.html:4`, `Resources/Private/Components/Atom/AspectRatio/AspectRatio.fluid.html:4`, `Resources/Private/Components/Atom/ScrollArea/ScrollArea.fluid.html:7`, `Resources/Private/Components/Layout/Section/Section.fluid.html:30`. Fix: drop the `style` arg in favour of named modifier props, or sanitise via a CSS-tokens regex.
- [MEDIUM] Editor `background_image` URL drops directly into `style="background-image: url(...)"` — `f:uri.image` returns a sanitised URL but is not htmlspecialchars-encoded inside the inline style; if an editor uploads a file with a single-quote in its name (FAL technically allows it on some storages), the CSS string breaks. Path: `ContentBlocks/ContentElements/testimonial-featured/templates/frontend.html:11`. Fix: htmlspecialchars + URL-encode (`-> f:format.htmlspecialchars()` after the image URI), or move to a `<img>` element with `object-fit:cover`.
- [LOW] Hard-coded fallback URLs in seed command — `https://example.com/desiderio/...` and `+43 1 555 010X` phone, `hello@example.com`. These are seed fixtures, not production, but the literals will land in `pages`/`tt_content` rows in any DB the command is run against. Path: `Classes/Command/SeedStyleguidePagesCommand.php:803,804,805,822,832`. Fix: gate the command behind a `--allow-non-dev` flag, or refuse to run when `Environment::getContext()->isProduction()`.
- [LOW] Seed command does not check execution context — `desiderio:styleguide:seed` is registered through Symfony `#[AsCommand]` so it is CLI-only by virtue of TYPO3's command bus (no HTTP route in `Configuration/Backend/Routes.php` or `Services.yaml`). Still, it accepts `--parent` (any pid) and writes to `pages`/`tt_content`/`sys_file_reference`/`sys_file_metadata`. Path: `Classes/Command/SeedStyleguidePagesCommand.php:74-149`. Fix: add `if (Environment::getContext()->isProduction()) { throw … }` and deny `--parent` values pointing into a production sysfolder via TSconfig allowlist.
- [LOW] Seed FAL writes are scoped to the *default* storage's root — `getStyleguideFalFolder()` calls `$storage->getRootLevelFolder(false)` on the default storage and creates `desiderio-styleguide/`. On most installs that is `fileadmin/`, but on multi-storage setups the "default" can be any mounted volume (incl. S3/SFTP). Path: `Classes/Command/SeedStyleguidePagesCommand.php:1628-1646`. Fix: accept `--storage` option, default to a documented fixture storage; refuse if storage is not local or `online=0`.
- [LOW] `scandir` + `Yaml::parseFile`/`json_decode` over `ContentBlocks/ContentElements/*/{config.yaml,fixture.json}` — Read-only and read from EXT path, but uses `is_readable` (TOCTOU) and `Yaml::parseFile` without any `Yaml::PARSE_…` safety flags. Practically unreachable from the web (only the CLI command + the FE-only `FixtureJsonViewHelper` page render call them), but worth noting if extension is published. Paths: `Classes/Command/SeedStyleguidePagesCommand.php:1711-1733`, `Classes/Data/StyleguideContentGroups.php:79-115,121-135`. Fix: cache parsed YAML at build time (already cached in-memory per request); document that `ContentBlocks/` must be read-only in production.
- [LOW] `FixtureJsonViewHelper` emits `<script id="styleguide-fixtures" type="application/json">` directly with `JSON_HEX_TAG` only — Good (`JSON_HEX_TAG` blocks `</script>`), but no nonce attribute and `escapeOutput = false`. Renders entire fixture content (ContentBlocks-shipped, not user data) as an inline JSON island. Path: `Classes/ViewHelpers/FixtureJsonViewHelper.php:18,32`. Fix: append a CSP nonce when available (`$GLOBALS['TYPO3_REQUEST']->getAttribute('nonce')`), or move fixtures to a static asset and `<script src=…>`.
- [LOW] `Resources/Public/Js/styleguide.js:135` — `previewFrame.innerHTML = renderPreview(data)` where `data` originates from the inline JSON above. The JSON is shipped in the extension, so the values are trust-bounded; but `renderPreview` calls `esc()` only on string-leaf values, not on object keys/structure. If a future fixture contributor adds tag-shaped values, prior `esc()` mitigates them. Path: `Resources/Public/Js/styleguide.js:135,535-540`. Fix: switch to `replaceChildren()` + `document.createElement` or a known-safe templating layer.
- [LOW] `ContentBlocks/ContentElements/chart-bar/templates/frontend.html:36` etc. — `getElementById('chart-bar-' + {data.uid})` inlines `{data.uid}` into JS context. `uid` is integer-typed so safe today, but pattern is fragile if anyone aliases `uid` to a non-integer in the future. Same as the [HIGH] inline-script CSP finding.
- [INFO] No SQL injection risk in `SeedStyleguidePagesCommand` — All 14 `where(...)` and `update(...)`/`set(...)` clauses use `createNamedParameter(...)` (with `ArrayParameterType::INTEGER` for IN-lists) and the `delete()`/`insert()`/`update()` calls go through Doctrine's high-level builder. No raw string interpolation found. References checked: lines 187-189, 232-234, 252, 270-271, 305-307, 330-334, 357, 382-385, 1659-1670, 1688-1693.
- [INFO] No CSRF surface — No FormController, no extbase actions, no Backend module routes, no AJAX endpoints registered. The Solr search form uses `s:searchForm` which is provided/CSRF-handled by EXT:solr.
- [INFO] No PSR-14 listeners or `$GLOBALS['TYPO3_CONF_VARS']['SC_OPTIONS']` hooks registered — `ext_localconf.php` is intentionally empty (just `defined('TYPO3') or die();`). Zero anonymous-class hook DoS surface in this extension.
- [INFO] No session/cookie usage in commands or VHs.
- [INFO] No PII logging — No `logger->info()/error()` calls anywhere in `Classes/`.
- [INFO] `tx_solr_suggest = PAGE` typeNum 7384 is publicly reachable — Configuration/Sets/Desiderio/setup.typoscript:8-29 exposes a JSON suggest endpoint (`additionalHeaders.10 = Content-type: application/json`), but it is the standard EXT:solr suggest with `no_cache=0`. Inherited surface; rate-limit at the web tier.

## Hardening recommendations (ranked)

1. Constrain editor-supplied iframe URLs (`map-embed`, `video-embed`, `feature-video`, `testimonial-video`) — add a TCA `Text` field with `eval=trim` plus a server-side allowlist via a small `processDatamap_postProcessFieldArray` PSR-14 listener (or a custom `eval` class). Pair with `Content-Security-Policy: frame-src https://www.youtube-nocookie.com https://player.vimeo.com …` in the site's middleware.
2. Replace inline `<script>` blocks in all 11 chart elements — extract to one ES module under `Resources/Public/Js/charts.js`, register via `f:asset.script` (so it inherits CSP nonce), and pass data only through the existing `data-chart-data` attributes (which already use `f:format.htmlspecialchars()`). This unblocks strict CSP and removes the `{data.uid}` JS-context interpolation pattern across the package.
3. Force-cast every editor numeric value before it touches a `style` attribute — `progress_value`, `overlay_opacity`, `height`, `percentage`, `gradient_from/to`. Either change TCA types (`Number`/`Color`/`Select`) or wrap with `f:format.number()` and a CSS-token regex filter ViewHelper. Same change closes the `Resources/Private/Components/Atom/{Skeleton,AspectRatio,ScrollArea}/Section` `style` pass-through risk.
4. Make the seed command production-safe — refuse when `Environment::getContext()->isProduction()`; require explicit `--allow-production` plus a documented FAL storage; default the Unsplash demo folder to a `_fixtures/` subfolder that the docs storage is configured for.
5. Document the htmlSanitizer dependency for News templates — the `f:format.html` on `newsItem.bodytext`/`teaser` (`Resources/Private/Extensions/News/Templates/News/Detail.html:74,87`, `Partials/List/Item.html:58-59`, `Templates/News/MagazineList.html:69-70`) only stays safe when the site enables TYPO3 14's HTML sanitizer. Add a guard in README.md and a functional test that asserts the sanitizer is on.
6. Restrict the `html` CType FluidStyledContent template — emit a backend warning if non-admin groups have access; ship a TSconfig snippet `TCEFORM.tt_content.CType.removeItems := addToList(html)` for editor groups.
7. Wrap the Solr debug partial render in a runtime guard — even though `SolrDefaults/setup.typoscript:6` sets `debugOutput = 0`, add `<f:if condition="{f:debug()} || {settings.debug}">` around the `f:format.raw` blocks at `Resources/Private/Solr/Partials/Result/Document.html:61` and `Resources/Private/Extensions/Solr/Partials/Result/Document.html:60`.
