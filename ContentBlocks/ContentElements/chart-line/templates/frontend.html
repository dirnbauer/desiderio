The line itself is drawn into .chart-line__canvas by Resources/Public/Js/charts.js (it reads data-chart-data). That SVG is decorative (aria-hidden): charts.js clears it before drawing, so any markup inside is lost and it carries no accessible name. Everything that conveys the data is server-rendered and stays meaningful without JS and for assistive technology: - a role="img" figure with a concise aria-label + an aria-describedby summary, - a visually-hidden (sr-only) data table carrying every period, value and unit. The editor maintains one human-readable "Label | value" list (data_points) for the summary and table; chart_data holds the matching JSON the line is drawn from. {data.unit -> f:or(alternative: '') -> f:format.trim()} {f:if(condition: '{data.series_label -> f:format.trim()}', then: '{data.series_label -> f:format.trim()}', else: '{f:translate(key: \'LLL:EXT:desiderio/ContentBlocks/ContentElements/chart-line/language/labels.xlf:aria.fallback\')}')} {f:if(condition: data.header, then: '{data.header}', else: '{seriesLabel}')} Count valid points and remember the first/last period so the summary states the span. {rowParts.0 -> f:or(alternative: '') -> f:format.trim()} {chartTitle}: line chart of {seriesLabel} {chartTitle}: {seriesLabel} trend across {pointCount} points, from {firstLabel} to {lastLabel}.
