# Security Audit (OWASP/Generic)

Date: 2026-05-03
Risk level: Low

## Summary
Desiderio is a presentation/theme TYPO3 extension with no HTTP routes, no auth flow, and no PSR-14 listeners or `SC_OPTIONS` hooks (`ext_localconf.php` is empty by design). Generic/OWASP risk is small: dependencies are current and pinned, no secrets are tracked, no `unserialize`/`eval`/dynamic-include surface. The only material generic findings are a third-party CDN loaded without Subresource Integrity, two npm transitive packages with lifecycle install scripts, and a permissive `composer` profile (`minimum-stability: dev`, no `secure-http` toggle). For TYPO3-specific (XSS, iframe, CSP) findings see `Documentation/Reports/typo3-security.md` — not duplicated here.

## Findings

- [MEDIUM] External CSS loaded over CDN without SRI / pinning (CWE-353, CWE-829) — `<f:asset.css href="https://cdn.hugeicons.com/font/hgi-stroke-rounded.css">` runs author-trust CSS on every page; no `integrity=`, no version pin. Compromise of `cdn.hugeicons.com` injects styles (and via `@import url(...)` could exfiltrate via `background-image:url(//attacker)`). Path: `Resources/Private/Templates/Layouts/Pages/Default.fluid.html:8`. Fix: vendor the icon CSS into `Resources/Public/Css/`, or pin `?v=<hash>` and add `integrity="sha384-…" crossorigin="anonymous"` plus `Content-Security-Policy: style-src 'self' cdn.hugeicons.com`.
- [MEDIUM] npm transitive deps with lifecycle install scripts (CWE-1357, supply chain) — `@parcel/watcher@2.5.6` and `msw@2.13.5` carry `hasInstallScript: true` per `package-lock.json` and run native binary download / postinstall code on `npm install`. Path: `package-lock.json` (entries `node_modules/@parcel/watcher`, `node_modules/msw`). Fix: run `npm ci --ignore-scripts` in CI/dev, or add `"ignore-scripts": true` to `.npmrc` and a `pre-build` step that explicitly opts-in. Also add `--audit-level=high` and consider `npm config set fund false`.
- [MEDIUM] Composer `minimum-stability: dev` (CWE-1357) — `composer.json:19` allows pulling unstable/dev releases on first install of a new transitive. `prefer-stable: true` mitigates but does not fully prevent it. Path: `composer.json:19`. Fix: set `"minimum-stability": "stable"` (the lock file already only contains stable releases, so no functional change), and add `"config": { "secure-http": true, "audit": { "abandoned": "fail" } }`.
- [MEDIUM] Editor-controlled object data interpolated into JS via `innerHTML` (CWE-79) — `previewFrame.innerHTML = renderPreview(data)`; `renderPreview` calls `esc()` on string-leaf values but concatenates `item.icon` and `d.icon` raw (intended for inline SVG). Today the data source is the in-repo, author-controlled `fixture.json`, but a future contributor adding `<img onerror>`-shaped icons or a server-side route that surfaces this VH would land XSS. Paths: `Resources/Public/Js/styleguide.js:135`, `Resources/Public/Js/styleguide.js:319,354`. Fix: build via `document.createElement` + `replaceChildren()`; render SVG via `DOMParser` with `image/svg+xml` and reject non-`svg` roots.
- [LOW] Historical `.mcp.json` retained in git history (CWE-540) — `.mcp.json` was tracked until commit `b54cdb38` (2026-04-26) before being removed and gitignored. Historical content was a local DDEV URL only — no token. Path: git history. Fix: acceptable as-is. If a real bearer/auth token is ever placed in `.mcp.json` in future, rotate it and `git filter-repo` the path.
- [LOW] `FixtureJsonViewHelper` emits inline `<script type="application/json">` with `escapeOutput=false` (CWE-79) — `JSON_HEX_TAG` correctly escapes `<`/`>` so `</script>` cannot break out, but an inline JSON island still violates a strict (`script-src 'self'`) CSP and is friction for future hardening. Path: `Classes/ViewHelpers/FixtureJsonViewHelper.php:18,32`. Fix: emit a CSP nonce attribute pulled from `$GLOBALS['TYPO3_REQUEST']->getAttribute('nonce')`, or move the JSON to a static file and reference via `<script src=…>`.
- [LOW] `shell_exec` in unit test invokes audit script (CWE-78, contained) — Test calls `shell_exec(escapeshellcmd(PHP_BINARY) . ' ' . escapeshellarg($script))`. Both args are escaped and the command/path are constants, so no injection in practice. Note that `escapeshellcmd` on `PHP_BINARY` is double-defence and not strictly needed. Path: `Tests/Unit/ContentElementAuditTest.php:38`. Fix: keep, or switch to `Symfony\Component\Process\Process` for cleaner argv-array semantics.
- [INFO] No `.env*`, credentials, `.pem`, `.key`, or `.p12` files exist or are tracked. `.gitignore` covers `.mcp.json`, `vendor/`, `node_modules/`, `var/`, `public/`, `composer.lock` (note: locking out `composer.lock` is unusual for an application but acceptable for a TYPO3 extension; root projects must regenerate). Path: `.gitignore:1-10`.
- [INFO] No `unserialize()`, `eval()`, `assert(string)`, or dynamic `require/include $var` in any PHP source. Verified across `Classes/`, `scripts/`, `Build/`, `Tests/`.
- [INFO] No subprocess execution in `Classes/` (production code). Only `Tests/Unit/ContentElementAuditTest.php` uses `shell_exec` with escaped constants.
- [INFO] No XXE risk — only `DOMDocument` use (`Build/Scripts/normalize-content-elements.php:936`) is a build-time script over project-internal XLIFF, not user input.
- [INFO] No FAL access takes user-controlled paths. `GeneralUtility::getFileAbsFileName('EXT:desiderio/...')` is always called with hardcoded extension paths (`Classes/Components/ComponentCollection.php:17`, `Classes/Data/StyleguideContentGroups.php:81,123`, `Classes/Command/SeedStyleguidePagesCommand.php:1610,1704`). `$folder->getFile($fileName)` in `SeedStyleguidePagesCommand.php:1617` uses `basename()` of an in-repo fixture path.
- [INFO] No SQL injection — all `Classes/Command/SeedStyleguidePagesCommand.php` queries use Doctrine `QueryBuilder` with `createNamedParameter` / `ArrayParameterType::INTEGER` (already documented in `typo3-security.md`).
- [INFO] No catastrophic-backtracking regexes in audit/seed scripts. Each alternation in `scripts/audit-content-elements.php:298` (`/(?<!var\()\b(?:#…|oklch\([^)]+\)|rgba?\([^)]+\)|hsla?\([^)]+\))/`) uses linear `[^)]+` greedy classes; the `[^"\']*` patterns at `:277` are likewise linear. No nested unbounded quantifiers found in `Build/Scripts/*.php` or `Classes/`.
- [INFO] All composer deps current: `typo3/cms-core v14.3.0`, `guzzlehttp/guzzle 7.10.0`, `symfony/* v7.4.x`, `doctrine/dbal 4.4.3`, `firebase/php-jwt v7.0.5`, `enshrined/svg-sanitize 0.22.0`. No abandoned packages, no `dev-`branch versions in lockfile, all resolved to `registry.npmjs.org` / `repo.packagist.org`.
- [INFO] `composer.json` `allow-plugins` is correctly minimal (3 trusted: `phpstan/extension-installer`, `typo3/class-alias-loader`, `typo3/cms-composer-installers`). No `"*": true`.
- [INFO] `package.json` declares only two scripts (`build:css`, `watch:css`); no `preinstall`/`postinstall`/`prepare` script in this project's own manifest.

## Hardening recommendations (ranked)

1. Vendor `cdn.hugeicons.com` CSS into `Resources/Public/Css/hugeicons.css` (one-time download + commit), or at minimum add `integrity="sha384-…"` and `crossorigin="anonymous"` to `Default.fluid.html:8` plus a CSP `style-src` allowlist; eliminates the only third-party origin loaded by Desiderio.
2. Add `.npmrc` with `ignore-scripts=true` and switch CI to `npm ci --ignore-scripts`; explicitly run `npm rebuild @parcel/watcher` only when needed (it is a transitive of `tailwindcss`/`shadcn` for native fsevents — the build still works without the native bin).
3. Tighten `composer.json`: change `minimum-stability` from `dev` to `stable`, add `"config": { "secure-http": true, "audit": { "abandoned": "fail" } }`, and run `composer audit` in CI alongside `composer validate --strict`.
4. Refactor `Resources/Public/Js/styleguide.js:135` away from `innerHTML`. Use `document.createDocumentFragment` + `createElement`, and isolate SVG injection through `DOMParser('image/svg+xml')` with a tag/attribute allowlist. Closes the latent XSS surface even if fixture authorship is loosened later.
5. Have `FixtureJsonViewHelper` (`Classes/ViewHelpers/FixtureJsonViewHelper.php:32`) read `$renderingContext->getRequest()->getAttribute('nonce')` and emit `<script nonce="...">`. Future-proofs Desiderio for a strict CSP `script-src 'self' 'nonce-…'`.
6. Add `composer audit` and `npm audit --omit=dev --audit-level=high` to CI (`.github/workflows/*` if present, else document in README) so dependency CVEs surface as build failures rather than during a pen test.
7. Rotate any token if `.mcp.json` ever held one. The historical commits only contained a `.ddev.site` URL, but document a "never commit .mcp.json" reminder in `.mcp.json.example`.
