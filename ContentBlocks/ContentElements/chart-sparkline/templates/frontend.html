The trend line is drawn into .chart-sparkline__svg by Resources/Public/Js/charts.js (it reads data-chart-json). That SVG is purely decorative (aria-hidden, focusable=false): charts.js clears it before drawing, so any markup placed inside would be lost and it carries no accessible name. Everything that conveys the numbers is server-rendered and stays meaningful without JS and for assistive technology: - a role="img" figure with a concise aria-label + an aria-describedby summary, - a visually-hidden (sr-only) data table listing every period, its value and the unit. The editor keeps one human-readable "Period | Value" list (data_points) for the summary and table; chart_data holds the matching JSON the sparkline is drawn from, so the picture and the table can never drift apart.
