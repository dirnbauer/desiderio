charts.js (initDonut) reads data-chart-data and DRAWS the donut ring + centre text into .chart-donut__canvas, wiping any static children. So the SVG is purely decorative (aria-hidden) and every meaning lives in server-rendered markup: a role="img" wrapper with a concise aria-label and an aria-describedby summary, plus a visually-hidden data table that carries the exact same numbers. data-chart-data is assembled from the same {data.segments} rows the table uses, so the picture and the table can never drift apart. Series colours come exclusively from the --chart-1..5 tokens. {f:if(condition: data.header, then: '{data.header}', else: '{f:translate(key: \'LLL:EXT:desiderio/ContentBlocks/ContentElements/chart-donut/language/labels.xlf:aria.fallback\')}')} {data.center_value -> f:or(alternative: '') -> f:format.stripTags() -> f:format.trim()} {data.center_label -> f:or(alternative: '') -> f:format.stripTags() -> f:format.trim()} [{"label":"{segment.label -> f:or(alternative: '') -> f:format.stripTags() -> f:format.trim()}","value":{segment.value -> f:or(alternative: '0') -> f:format.trim()}},]
