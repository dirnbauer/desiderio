The coloured wedges are drawn into .chart-pie__canvas by Resources/Public/Js/charts.js (it reads data-chart-data). That SVG is decorative (aria-hidden): charts.js clears it before drawing, so any markup placed inside it is lost and it carries no accessible name. Everything that conveys the data is server-rendered and stays meaningful without JS and for assistive technology: - a role="img" figure with a concise aria-label + an aria-describedby summary, - a visible, named legend keyed to the same --chart-N tokens charts.js paints with, - a visually-hidden (sr-only) data table carrying every label, value, unit and share of the whole. The editor maintains one human-readable "Label | Value" line per segment (segments); chart_data holds the matching JSON the canvas reads. Each row is split on the pipe; rows are split on the newline. {data.palette -> f:or(alternative: 'default')} {data.chart_size -> f:or(alternative: 'medium')} {data.unit -> f:or(alternative: '') -> f:format.trim()} {f:if(condition: data.header, then: '{data.header}', else: '{f:translate(key: \'LLL:EXT:desiderio/ContentBlocks/ContentElements/chart-pie/language/labels.xlf:aria.fallback\')}')} Pass 1: count segments and total their values so the table can show share of the whole. {parts.0 -> f:or(alternative: '') -> f:format.trim()} {parts.1 -> f:or(alternative: '0') -> f:format.trim()} {chartTitle}: pie chart with {segmentCount} segments.
