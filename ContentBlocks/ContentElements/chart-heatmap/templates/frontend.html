The coloured intensity grid is drawn into .chart-heatmap__plot by Resources/Public/Js/charts.js (it reads data-chart-json). That grid is decorative (aria-hidden): charts.js clears __plot before drawing, so any markup placed inside it is lost and it carries no accessible name. Everything that conveys the data is server-rendered and stays meaningful without JS and for assistive technology: - a role="img" figure with a concise aria-label + an aria-describedby summary, - a visually-hidden (sr-only) data table carrying every row, column, value and unit. The editor maintains one human-readable "Row label | v1, v2, v3" list (series) plus the column labels (col_labels); chart_data holds the matching JSON the grid reads. {data.unit -> f:or(alternative: '') -> f:format.trim()} {f:if(condition: data.header, then: '{data.header}', else: '{f:translate(key: \'LLL:EXT:desiderio/ContentBlocks/ContentElements/chart-heatmap/language/labels.xlf:aria.fallback\')}')} Count non-empty rows so the summary can state the grid size. {chartTitle}: heatmap of {rowCount} rows by {colCount} columns.
