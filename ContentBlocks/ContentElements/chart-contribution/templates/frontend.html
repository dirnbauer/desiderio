charts.js (initContribution) reads data-chart-json and data-unit, then DRAWS the bars, y-axis scale and x-axis ticks into .chart-contribution__svg, wiping any static children. So the SVG is purely decorative (aria-hidden) and every meaning lives in server-rendered markup: a role="img" wrapper with a concise aria-label + an aria-describedby summary, and a visually-hidden data table that carries the exact same numbers. data-chart-json is assembled from the same {data.points} rows the table uses, so the picture and the table can never drift apart. {data.metric_label -> f:or(alternative: '') -> f:format.stripTags() -> f:format.trim()} {data.unit -> f:or(alternative: '') -> f:format.stripTags() -> f:format.trim()} {f:if(condition: data.header, then: '{data.header}', else: '{f:translate(key: \'LLL:EXT:desiderio/ContentBlocks/ContentElements/chart-contribution/language/labels.xlf:aria.fallback\')}')} {f:if(condition: metricLabel, then: '{metricLabel}', else: '{headerTitle}')} [{"label":,"value":},]
